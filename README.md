# Русик, наливай!

## Что это

`Русик, наливай!` - статическое HTML/PWA-приложение для Android: закрытая party-игра для компании друзей в формате "Правда / Действие / Я никогда не / Мини-игры / Штрафы / Дуэли".

Проект сделан на чистых HTML, CSS и JavaScript без сборщика, фреймворков, серверной части и внешних CDN. После первого открытия PWA кэширует app shell через service worker и может работать офлайн.

Текущая база содержит 1000 карточек:

```text
Действие - 600
Правда - 150
Я никогда не - 70
Мини-игры и дуэли - 150
Штраф / хаос - 30
```

## Как запустить локально

Откройте папку проекта и запустите простой статический сервер:

```bash
python -m http.server 8080
```

Затем откройте:

```text
http://localhost:8080
```

Также можно использовать Node.js:

```bash
npx serve .
```

## Как опубликовать через GitHub Pages

1. Создать репозиторий `rusik-nalivay` на GitHub.
2. Загрузить все файлы проекта в ветку `main`.
3. Открыть `Settings`.
4. Перейти в `Pages`.
5. В разделе `Build and deployment` выбрать:
   `Source: Deploy from a branch`
   `Branch: main`
   `Folder: /root`
6. Сохранить.
7. Через несколько минут GitHub выдаст ссылку на сайт.
8. Открыть ссылку на Android в Chrome.
9. Нажать меню Chrome.
10. Выбрать "Добавить на главный экран".

Все пути в проекте относительные (`./styles.css`, `./app.js`, `./manifest.json`), поэтому приложение корректно работает в подпапке GitHub Pages.

## Как установить на Android

1. Откройте опубликованный сайт в Chrome на Android.
2. Дождитесь первой загрузки приложения.
3. Откройте меню Chrome.
4. Нажмите "Добавить на главный экран" или "Установить приложение".
5. Запустите установленную PWA с главного экрана.

После первого запуска app shell доступен офлайн. История карточек хранится локально на устройстве.

## Как обновлять карточки

Карточки лежат в `cards.js` в объекте:

```js
const GAME_CARDS = {
  action: [],
  truth: [],
  never: [],
  mini_game: [],
  chaos: []
};
```

Каждая карточка должна иметь:

```js
{
  id: "action_001",
  type: "action",
  title: "Действие",
  timer: null,
  text: "Текст карточки"
}
```

Поддерживаются переменные:

```text
{player}
{randomPlayer}
{randomPlayer2}
{allPlayers}
```

В `cards.js` есть ручная стартовая база и генераторы, которые расширяют её до полной тысячи карточек при загрузке. Если нужны полностью ручные карточки, можно постепенно заменять сгенерированные записи обычными объектами в нужных массивах. Менять `app.js` для этого не нужно: достаточно сохранять уникальные `id`, правильный `type` и непустой `text`.

```text

├── index.html
├── styles.css
├── app.js
├── cards.js
├── manifest.json
├── service-worker.js
├── README.md
├── sounds/
│   └── .gitkeep
└── icons/
    ├── icon-192.png
    └── icon-512.png
```

Папка `sounds/` оставлена для будущего файла `timer-end.mp3`. Сейчас сигнал окончания таймера реализован через Web Audio API в `app.js`, поэтому приложение не зависит от внешнего аудиофайла.

## Где хранится история игры

История хранится в `localStorage` браузера на устройстве:

```text
usedCards
usedChaosCards
selectedPlayers
gameStats
```

Это позволяет не повторять карточки до окончания базы и сохранять выбранных игроков между запусками.

## Как сбросить историю

В игре откройте `Меню` и нажмите `Сбросить историю карточек`. Приложение покажет подтверждение:

```text
Точно сбросить историю? Уже выпавшие карточки снова смогут появляться.
```

После подтверждения история выпавших карточек и статистика очищаются, а выбранные игроки остаются.
