# Русик, наливай!

Закрытая party-игра для компании друзей.

## Что внутри

- 300 карточек
- 150 действий
- 60 карточек “Правда”
- 40 карточек “Я никогда не”
- 35 мини-игр
- 15 штрафных chaos-карточек

## Возможности

- Выбор игроков перед стартом
- Добавление новых игроков
- Удаление добавленных игроков
- Сохранение новых игроков в localStorage
- Склонения имён для карточек
- Карточки не повторяются до окончания базы
- Таймеры для заданий
- Звук и вибрация по окончании таймера
- PWA-режим
- Иконка приложения для Android
- Добавление игры на главный экран телефона

## Режимы игры

Перед стартом можно выбрать режим:

- Все карточки
- Правда или действие
- Только правда
- Я никогда не...
- Мини-игры
- Хаос
- Свой набор

В режиме “Свой набор” можно вручную выбрать типы карточек.

Кнопка “Отказ / штраф” работает во всех режимах и использует chaos-карточки как штрафные.

## Очередь игроков

Активные игроки получают карточки по очереди, а не случайно.

Если выбраны Русик, Инна, Лера и Саша, ход идет так:

```text
Русик -> Инна -> Лера -> Саша -> снова Русик
```

В парных карточках активный игрок идет по очереди, а второй игрок выбирается по правилам пола и ограничениям Леры.

Если игрок нажимает “Отказ / штраф”, штрафная карточка применяется к тому же игроку.

## Обновление карточек

Карточки переписаны в более прямом и конкретном стиле.

Каждая карточка стала понятнее: кто выполняет, что делает, с кем выполняет, сколько секунд и какой штраф за отказ.

На каждой карточке активный игрок сначала делает игровой глоток перед выполнением.

Блок “Правда” усилен: вопросы стали прямее, личнее и разнообразнее. Карточки не должны повторять один и тот же смысл разными словами.

## Как запустить локально

Откройте папку проекта и запустите простой статический сервер:

```bash
python -m http.server 8080
```

Затем откройте:

```text
http://localhost:8080
```

## Локальные данные

В браузере сохраняются:

```text
usedCards
usedChaosCards
selectedPlayers
customPlayers
activePlayerTurnIndex
gameStats
```

Это позволяет не повторять карточки до окончания базы, хранить выбранных игроков и сохранять добавленных игроков между запусками.
